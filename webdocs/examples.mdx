# Examples

This guide provides comprehensive, copy-paste runnable examples for all AIKit features. Each example includes prerequisites, step-by-step instructions, and expected outputs.

## Prerequisites

Before running these examples, ensure you have:

### Required Setup

```bash
# 1. Install AIKit (if not already installed)
cargo install aikit

# 2. Set up GitHub token (required for most operations)
export GITHUB_TOKEN="your_github_personal_access_token"

# 3. Verify installation
aikit --version
```

### Optional Tools

```bash
# Install Claude Code (for Claude examples)
npm install -g @anthropic-ai/claude-code

# Install other AI agent CLIs as needed
# Each agent has different installation requirements
```

## Advanced Integration Examples

### 6. Multi-Agent Project Setup

**Goal**: Configure a project for multiple AI agents.

**Prerequisites**: Multiple AI agent CLIs installed

**Step 1: Initialize project**
```bash
aikit init multi-agent-project
cd multi-agent-project
```

**Step 2: Configure multiple agents in aikit.toml**
```toml
# aikit.toml
version = "1.0"
install_dir = ".aikit"

[agents.claude]
name = "Claude Code"
key = "claude"
folder = "claude"
requires_cli = true
output_dir = "commands/claude"
extensions = ["md", "py", "js", "rs"]

[agents.gemini]
name = "Google Gemini"
key = "gemini"
folder = "gemini"
requires_cli = false
output_dir = "commands/gemini"
extensions = ["md", "py", "js", "ts"]

[preferences]
default_agent = "claude"
verbose = true
```

**Step 3: Install packages for different agents**
```bash
# Install Claude-specific package
aikit install https://github.com/example/claude-tools --agent claude

# Install general-purpose package
aikit install https://github.com/example/data-tools

# Install Gemini-specific package
aikit install https://github.com/example/gemini-extensions --agent gemini
```

**Step 4: Verify setup**
```bash
aikit list --detailed
aikit check
```

**Expected output**:
```
Multi-agent project configured successfully
- Claude: 3 packages installed
- Gemini: 2 packages installed
- General: 1 package installed
- Total: 6 packages across 2 agents
```

### 7. Package Development Workflow

**Goal**: Create, build, and publish an AIKit package.

**Prerequisites**: GitHub repository, GitHub token

**Step 1: Initialize package**
```bash
# Create package directory
mkdir my-ai-package
cd my-ai-package

# Initialize Git repository
git init
git remote add origin https://github.com/yourusername/my-ai-package.git

# Initialize AIKit package
aikit package init "my-ai-package" --description "Useful AI agent tools"
```

**Step 2: Configure package**
```toml
# Edit aikit.toml
name = "my-ai-package"
version = "0.1.0"
description = "Useful AI agent tools"
author = "Your Name"

[agents.claude]
commands = ["analyze", "generate", "optimize"]

[agents.gemini]
commands = ["analyze", "create"]

[dependencies]
# Add package dependencies if needed
```

**Step 3: Add package content**
```bash
# Create command files
mkdir -p commands/claude commands/gemini

# Create Claude commands
cat > commands/claude/analyze.md << 'EOF'
# analyze

Analyze code or text for improvements.

## Usage
claude analyze <target>

## Examples
claude analyze src/main.rs
claude analyze --format json src/
EOF

# Create Gemini commands
cat > commands/gemini/create.md << 'EOF'
# create

Create new content or code.

## Usage
gemini create <type> <name>

## Examples
gemini create component Button
gemini create function validateInput
EOF
```

**Step 4: Build package**
```bash
# Build the package
aikit package build

# Check build output
ls -la dist/
```

**Expected build output**:
```
dist/
‚îî‚îÄ‚îÄ my-ai-package-0.1.0.zip
```

**Step 5: Test package locally**
```bash
# Install locally for testing
aikit install ./dist/my-ai-package-0.1.0.zip --local

# Test installation
aikit list | grep my-ai-package

# Run a command
aikit run my-ai-package analyze --help
```

**Step 6: Publish package**
```bash
# Commit and push changes
git add .
git commit -m "Initial package release"
git push origin main

# Create GitHub release
aikit release v0.1.0

# Publish to registry
aikit package publish
```

**Expected publish output**:
```

‚úì Uploaded package: my-ai-package-0.1.0.zip
‚úì Package published to registry
‚úì Available for installation: aikit install my-ai-package
```

### 8. CI/CD Integration

**Goal**: Set up automated package building and publishing in CI/CD.

**Prerequisites**: GitHub repository with Actions enabled

**GitHub Actions workflow (.github/workflows/release.yml)**:
```yaml
name: Release AIKit Package

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v4

    - name: Set up Rust
      uses: dtolnay/rust-toolchain@stable

    - name: Install AIKit
      run: cargo install aikit

    - name: Configure GitHub token
      run: echo "GITHUB_TOKEN=${{ secrets.GITHUB_TOKEN }}" >> $GITHUB_ENV

    - name: Build package
      run: aikit package build

    - name: Create release
      run: aikit release ${{ github.ref_name }}

    - name: Publish package
      run: aikit package publish
```

**Alternative: Simple build workflow**
```yaml
name: Build Package

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v4

    - name: Install AIKit
      run: cargo install aikit

    - name: Build package
      run: aikit package build

    - name: Upload artifacts
      uses: actions/upload-artifact@v4
      with:
        name: aikit-package
        path: dist/*.zip
```

### 9. Custom Agent Integration

**Goal**: Add support for a custom AI agent.

**Prerequisites**: Custom agent with CLI or API

**Step 1: Define agent configuration**
```toml
# Add to aikit.toml
[agents.myagent]
name = "My Custom Agent"
key = "myagent"
folder = "myagent"
requires_cli = true
install_url = "https://myagent.ai/install"
output_dir = "commands/myagent"
arg_placeholder = "<arg>"
extensions = ["md", "txt", "py", "js"]
output_format = "markdown"
```

**Step 2: Create agent-specific commands**
```bash
# Create command directory
mkdir -p commands/myagent

# Create command files
cat > commands/myagent/help.md << 'EOF'
# help

Get help and usage information.

## Usage
myagent help [topic]

## Examples
myagent help
myagent help commands
EOF

cat > commands/myagent/analyze.md << 'EOF'
# analyze

Analyze code or documents.

## Usage
myagent analyze <file> [options]

## Options
- `--format`: Output format (text, json, xml)
- `--depth`: Analysis depth (shallow, medium, deep)

## Examples
myagent analyze src/main.py
myagent analyze --format json --depth deep report.md
EOF
```

**Step 3: Test integration**
```bash
# Validate configuration
aikit check agents

# Test command execution
aikit run myagent help

# List available commands
aikit list myagent
```

**Expected output**:
```
‚úì Agent 'myagent' configured successfully
‚úì CLI detected: myagent command available
‚úì Commands: help, analyze
‚úì Integration test passed
```

### 10. Troubleshooting Examples

**Goal**: Diagnose and fix common AIKit issues.

#### Issue: Package Installation Fails

**Symptoms**: `aikit install` returns authentication error

**Solution**:
```bash
# Check GitHub token
echo $GITHUB_TOKEN

# Test token validity
curl -H "Authorization: token $GITHUB_TOKEN" https://api.github.com/user

# Reconfigure token
export GITHUB_TOKEN="new_token_here"
aikit install https://github.com/example/package
```

#### Issue: Agent CLI Not Found

**Symptoms**: `aikit check` shows missing CLI

**Solution**:
```bash
# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Verify installation
which claude
claude --version

# Re-run check
aikit check
```


#### Issue: Configuration Validation Errors

**Symptoms**: AIKit reports invalid configuration

**Solution**:
```bash
# Validate configuration
aikit config validate

# Show detailed errors
aikit config validate --verbose

# Create default configuration
aikit config init

# Edit configuration
nano aikit.toml
```

#### Issue: Build Fails

**Symptoms**: `aikit package build` fails

**Solution**:
```bash
# Check package structure
ls -la
cat aikit.toml

# Validate package configuration
aikit package build --dry-run

# Build with debug output
aikit package build --debug

# Check for missing files
find . -name "*.md" -path "./commands/*"
```

## Performance Optimization Examples

### 11. Large Project Configuration

**Goal**: Optimize AIKit for large projects with many packages.

```toml
# aikit.toml - Large project optimization
version = "1.0"
install_dir = ".aikit"

[registry]
# Shorter cache TTL for development
cache_ttl_minutes = 15
cache_dir = ".aikit/cache"

[preferences]
# Enable verbose output for debugging
verbose = true
# Disable confirmations for automation
confirm_overwrite = false
# Longer timeout for large operations
network_timeout = 120

[build]
# Optimize build settings
output_dir = "dist"
include_sources = false  # Smaller packages
compress = true
target_agents = ["claude", "gemini"]  # Build only for used agents
```

### 12. Team Collaboration Setup

**Goal**: Configure AIKit for team development.

**Global configuration (~/.aikit/config.toml)**:
```toml

confirm_overwrite = true

[registry]
```

**Project configuration (aikit.toml)**:
```toml

```
