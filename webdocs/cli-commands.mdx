# CLI Commands

AIKit provides a comprehensive command-line interface for managing AI agent packages, templates, and development workflows. This guide covers all available commands with detailed usage examples.

## Command Overview

AIKit organizes functionality into logical command groups:

### Core Commands
- **`init`** - Initialize new Spec-Driven Development projects
- **`agent run`** - Run a coding agent with a prompt (supports streaming events)
- **`agent list`** - List project-scoped agent definitions
- **`mcp install`** / **`mcp register`** - Self-register the `aikit` binary into an agent's MCP config file
- **`mcp list`** - List supported agents and their project/global config paths
- **`agent mcp`** - List supported agents or merge one MCP server into agent config files
- **`serve`** - Start an HTTP server for multi-turn agent sessions ([dedicated page](/serve))
- **`session sync`** - Sync raw, secret-scrubbed Claude Code / Codex session transcripts to S3-compatible blob storage
- **`llm`** - Invoke an LLM via OpenAI-compatible API (supports streaming and JSON output)
- **`check`** / **`agent check`** - Validate installed tools and AI agent CLIs
- **`version`** - Display version information

### Package Management
- **`install`** - Install packages from GitHub URLs
- **`update`** - Update installed packages to latest versions
- **`remove`** - Remove installed packages
- **`list`** - Display installed packages

### Package Development
- **`package init`** - Initialize new package with aikit.toml
- **`package build`** - Build package for distribution
- **`package publish`** - Publish package to registry

### Release Management
- **`release`** - Create GitHub releases with package files

## Global Options

All commands support these global options:

```bash
# Enable debug output for troubleshooting

# Display version information
aikit --version

# Show help for any command
```


## Command Details

### init

Initialize a new Spec-Driven Development project with AI agent integration.

```bash
# Create new project in subdirectory
aikit init my-project

# Initialize in current directory
aikit init --here

# Initialize for specific AI agent
aikit init my-project --ai claude

# Skip Git initialization
aikit init my-project --no-git

# Force initialization in non-empty directory
aikit init existing-dir --force
```

**Options:**
- `--ai <AGENT>` - Target AI assistant (see supported agents below)
- `--script <TYPE>` - Script type (sh or ps)
- `--here` - Initialize in current directory
- `--force` - Skip confirmation for non-empty directories
- `--no-git` - Skip Git repository initialization
- `--github-token <TOKEN>` - GitHub token for API requests
- `--skip-tls` - Skip TLS verification (not recommended)
- `--ignore-agent-tools` - Skip CLI validation for selected agent

**Supported AI Agents:**
AIKit supports 18 AI assistants across different categories:

**Primary Assistants:**
- `claude` - Claude Code by Anthropic
- `cursor` - Cursor AI IDE
- `copilot` - GitHub Copilot
- `gemini` - Google Gemini

**Additional Assistants:**
- `qwen` - Qwen Code
- `opencode` - OpenCode
- `codex` - GitHub Codex
- `windsurf` - Windsurf IDE
- `kilocode` - KiloCode
- `auggie` - Auggie CLI
- `roo` - Roo Code
- `codebuddy` - CodeBuddy CLI
- `qoder` - Qoder CLI
- `amp` - Amp
- `shai` - Shai
- `continue` - Continue (Codex)
- `bob` - Bob

**What it creates:**
- Project directory with basic structure
- `aikit.toml` configuration file
- `.aikit/` directory for internal state
- Git repository (unless `--no-git` specified)
- Agent-specific templates and scripts

**Example output:**

### agent run

Run a coding agent with a prompt. Executes AI coding agents directly from
the CLI, supporting buffered output, agent-native streaming, and structured
NDJSON event delivery.

**Runnable agents:** `codex`, `claude`, `gemini`, `opencode`, `agent`,
`aikit` (built-in), `auto`

```bash
# Run an agent with an inline prompt
aikit agent run --agent claude -p "Refactor this function for clarity"

# Read prompt from stdin
echo "Add error handling to main.rs" | aikit agent run --agent opencode

# Enable agent-native streaming output
aikit agent run --agent claude --stream -p "Summarize the project"

# Emit structured NDJSON events to stdout
aikit agent run --agent claude --events -p "Summarize the project"

# Combine --events and --stream for streaming-aware JSON output
aikit agent run --agent claude --events --stream -p "Refactor this module"
```

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--agent` | `-a` | Runnable agent key | **Required** |
| `--model` | `-m` | Model passed to the agent | Omitted unless `-m` is set; agent binary defines behavior |
| `--prompt` | `-p` | Prompt to run | Reads from stdin if omitted |
| `--yolo` | | Auto-confirm, skip checks | `false` |
| `--stream` | | Enable agent-native streaming output flags | `false` |
| `--events` | | Emit NDJSON event stream to stdout | `false` |
| `--progress` | | Display live human-readable progress on stderr (conflicts with `--events`) | `false` |
| `--debug` | | Verbose diagnostic output (global `aikit` flag; shown on `run` help) | `false` |

`-a` / `--agent` is required. `aikit agent run` does not use `CODING_AGENT` or `CODING_AGENT_MODEL`. Without `-m`, no model is passed through to the agent.

#### `--stream` vs `--events`

These flags are independent and can be combined:

- **`--stream`**: Tunes agent-native partial output flags (e.g. `--stream-partial-output`, `stream-json`). This is passed through as `RunOptions::stream` to the argv builder. The CLI output format is unchanged.
- **`--events`**: Switches the CLI to call `run_agent_events` and emit one JSON object per line (NDJSON) to stdout. Incremental delivery; process exit code matches child exit when no SDK error occurs.
- **Combined (`--events --stream`)**: Uses events-mode JSON output AND adds stream-partial flags for supported agents.

#### NDJSON event format

When using `--events`, each line of stdout is a JSON object:

```json
{"agent_key":"claude","seq":1,"stream":"stdout","payload":{"json_line":{"type":"progress","message":"Starting..."}}}
{"agent_key":"claude","seq":2,"stream":"stdout","payload":{"raw_line":"Processing file..."}}
{"agent_key":"claude","seq":3,"stream":"stderr","payload":{"raw_line":"Warning: large context"}}
```

Fields:
- `agent_key` - The agent that produced the event
- `seq` - Monotonically increasing sequence number
- `stream` - Source stream: `"stdout"` or `"stderr"`
- `payload` - One of: `{"json_line": <object>}`, `{"raw_line": "<string>"}`, or `{"raw_bytes": "<base64>"}`

See [aikit-sdk README](../aikit-sdk/README.md) for complete streaming API documentation.

Run `aikit agent run --help` for the authoritative option reference.

### serve

Start an HTTP server that exposes the agent runtime over a small REST + SSE
surface. Sessions are created implicitly on the first `POST /api/v1/messages` that
omits `session_id`; subsequent calls resume by passing the returned id.

```bash
# Defaults: 127.0.0.1:8787, 300s run timeout, 10 concurrent runs
aikit serve

# Bind publicly, require an API key, override the timeout
aikit serve --host 0.0.0.0 --port 8787 \
  --run-timeout-secs 600 --max-sessions 20 \
  --api-key "$(openssl rand -hex 32)"
```

**Options:**

| Flag | Description | Default |
|------|-------------|---------|
| `--host` | Bind address | `127.0.0.1` |
| `--port` | TCP port | `8787` |
| `--run-timeout-secs` | Abort agent runs that exceed this many seconds | `300` |
| `--max-sessions` | Maximum concurrent active runs (429 over the cap) | `10` |
| `--api-key` | Require `Authorization: Bearer <key>` on every request. Also reads `AIKIT_SERVE_API_KEY`. | unset |

**Endpoints:** `GET /healthz`, `GET /readyz`, `GET /api/v1/agents`, `POST /api/v1/messages`,
`GET /api/v1/sessions`, `GET /api/v1/sessions/{id}`, `DELETE /api/v1/sessions/{id}`.
`GET /api/` redirects `308` to `/api/v1`.

**Choosing a response shape on `/api/v1/messages`** — content negotiation via
`Accept`:

```bash
# SSE (incremental events; default if Accept is missing or */*)
curl -sN -X POST http://127.0.0.1:8787/api/v1/messages \
  -H 'Accept: text/event-stream' \
  -H 'Content-Type: application/json' \
  -d '{"agent":"aikit","content":"Say hello."}'

# Single JSON body (runs to completion, returns assistant text + session_id)
curl -s -X POST http://127.0.0.1:8787/api/v1/messages \
  -H 'Accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{"agent":"aikit","content":"Say hello."}' | jq .
```

Any other explicit `Accept` returns `406 Not Acceptable`. See the
[serve API reference](/serve) for full request/response schemas, SSE event
names, session lifecycle, and error codes.

### session sync

Sync **raw, secret-scrubbed** session transcripts from local AI coding tools to
S3-compatible blob storage. Sync is a **write-only producer**: it reads the
JSONL files each tool already wrote to disk, scrubs credentials, and uploads
content-addressed snapshots plus a JSON metadata sidecar. It never launches the
tool and never reads back — analysis/search is a separate downstream tool.

Only the two flat-file JSONL tools are supported today: **Claude Code** and
**Codex**. OpenCode (SQLite) is out of scope for v1.

Every object lands under a deterministic key inside the bucket:

```
<key-prefix>/<owner>/<tool>/<session-id>/<sha256>.jsonl       # scrubbed transcript
<key-prefix>/<owner>/<tool>/<session-id>/<sha256>.meta.json   # envelope: host, hash, timestamps, versions
```

```bash
# Credentials come from the standard AWS environment, never flags.
export AWS_ACCESS_KEY_ID=... AWS_SECRET_ACCESS_KEY=...

# One-shot: sync all detected Claude Code sessions to a MinIO bucket
aikit session sync \
  --owner alice \
  --bucket my-sessions \
  --endpoint https://minio.internal:9000 \
  --tool claude_code

# Preview only — detect, scrub, hash, summarize; upload nothing
aikit session sync --owner alice --bucket my-sessions \
  --endpoint https://minio.internal:9000 --dry-run --format json

# Continuously watch and sync on every change (laptop daemon / pod sidecar)
aikit session sync --owner alice --bucket my-sessions \
  --endpoint https://minio.internal:9000 --watch
```

**Options:**

| Flag | Env | Default | Description |
|------|-----|---------|-------------|
| `--bucket` | `AIKIT_SYNC_BUCKET` | — (required) | Target S3 bucket |
| `--endpoint` | `AIKIT_SYNC_ENDPOINT` | — (required) | S3-compatible endpoint URL |
| `--region` | `AIKIT_SYNC_REGION` | `us-east-1` | Region (a sentinel value is fine for MinIO) |
| `--owner` | `AIKIT_SYNC_OWNER` | credential-derived | Owner prefix; required (see fail-closed rule below) |
| `--key-prefix` | `AIKIT_SYNC_PREFIX` | `sessions/` | Root key prefix — the top-level "folder" in the bucket |
| `--tool` | — | all JSONL tools | Repeatable; `claude_code` or `codex` |
| `--watch` | — | off | Continuous mode: sync on filesystem change |
| `--dry-run` | — | off | Detect + scrub + hash + summarize, upload nothing |
| `--allow-http` | `AIKIT_SYNC_ALLOW_HTTP` | `false` | Permit plain-HTTP endpoints (local MinIO dev) |
| `--format` | — | `default` | `default` (human) or `json` (machine-readable summary) |
| `--log-level` | `RUST_LOG` | `info` | Log verbosity |

**Credentials** are read from the standard AWS environment
(`AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY`, or a workload-identity chain) and
are **never** accepted as flags. Two additional env-only knobs:

- `AIKIT_SYNC_CREDENTIAL_OWNER` — the owner identity derived from the credential.
  If it is set **and** disagrees with `--owner`, sync **fails closed** (exit `2`)
  rather than guessing. If neither is set, sync also refuses to start — an owner
  is always required.
- `AIKIT_SYNC_ENDPOINT_CA_BUNDLE` — path to a custom CA PEM bundle for a
  self-signed MinIO/S3 endpoint.

**Idempotency & retention:** the object key is the SHA-256 of the scrubbed
bytes, so re-syncing unchanged content is a no-op; a grown transcript produces a
new version object and **all versions are retained** (v1 keeps everything).

**Exit codes:** `0` everything synced or already current · `1` one or more files
failed after retries · `2` configuration/auth error before any upload (missing
bucket/endpoint, or an owner mismatch).

Requires a build with the `agent-adapters` feature (plus `watcher` for `--watch`).

### agent mcp

Register MCP servers in agent-specific config files (JSON or TOML). Six
catalog keys are supported: `cursor-agent`, `claude`, `gemini`, `copilot`,
`opencode`, `codex` (aliases: `cursor`, `vscode`). See
[MCP registration](/mcp) for paths, Rust (`aikit-sdk`), and Python
(`aikit-py`) equivalents.

```bash
# Agents and config paths (project vs global)
aikit agent mcp list

# Stdio server: repeat --arg for each argv token; repeat --env KEY=value for env
aikit agent mcp add --agent claude --scope project --project . --name fs \
  --command npx --arg -y --arg @modelcontextprotocol/server-filesystem --arg .

# HTTP server: repeat --header KEY=value for headers
aikit agent mcp add --agent gemini --scope project --project . --name api \
  --url http://127.0.0.1:8730/mcp
```

| Flag | Description |
| --- | --- |
| `--agent` | Catalog key or alias (`cursor`, `vscode`) |
| `--scope` | `project` or `global` |
| `--project` | Project root when scope is project (default `.`) |
| `--name` | Server id inside `mcpServers` / `servers` / `mcp` / Codex table |
| `--command` / `--url` | Stdio executable or HTTP URL (exactly one required) |
| `--arg` | One argv token for stdio (repeat) |
| `--env` | `KEY=value` for stdio env (repeat) |
| `--header` | `KEY=value` for HTTP headers (repeat) |
| `--overwrite` | Replace existing entry with the same name |

### mcp install / mcp register

Self-register the `aikit` binary into any supported agent's MCP config file. The binary path is resolved automatically from `current_exe()`. `mcp register` is an alias for `mcp install`.

```bash
# Preview without writing (dry-run)
aikit mcp install --agent cursor --stdio --dry-run

# Write .cursor/mcp.json in the current directory
aikit mcp install --agent cursor --stdio --scope project --overwrite

# Write ~/.claude.json (global scope)
aikit mcp install --agent claude --stdio --scope global --overwrite

# Use alias
aikit mcp register --agent cursor --stdio --dry-run
```

| Flag | Required | Default | Description |
|------|----------|---------|-------------|
| `--agent` | Yes | — | Agent key or alias (`cursor`, `claude`, `gemini`, `copilot`/`vscode`, `opencode`, `codex`) |
| `--stdio` | Yes (or `--url`) | `false` | Stdio transport; auto-generates command from `current_exe()` |
| `--url` | Yes (or `--stdio`) | — | HTTP transport URL |
| `--scope` | No | `project` | `project` (CWD) or `global` (home dir) |
| `--name` | No | `aikit` | Server entry name |
| `--overwrite` | No | `false` | Replace an existing entry with the same name |
| `--dry-run` | No | `false` | Print intended config; do not write any file |
| `--arg` | No | — | Override argv tokens (repeatable) |
| `--env` | No | — | `KEY=value` env vars for stdio (repeatable) |

### llm

Invoke an LLM via an OpenAI-compatible API. Supports both streaming and non-streaming requests, JSON output format, and usage tracking.

```bash
# Simple prompt (uses OPENAI_API_KEY env var by default)
aikit llm -m gpt-4o -p "Explain Rust ownership in one paragraph"

# Read prompt from a file
aikit llm -m gpt-4o --prompt-file prompt.txt

# Stream output token by token
aikit llm -m gpt-4o -p "Write a haiku about coding" --stream

# JSON output format
aikit llm -m gpt-4o -p "What is 2+2?" --format json

# Write output to a file
aikit llm -m gpt-4o -p "Summarize this" -o summary.txt

# Use a custom API endpoint
aikit llm -m my-model -u https://my-llm.example.com/v1 -p "Hello"

# Suppress non-essential output
aikit llm -m gpt-4o -p "Quick answer" --quiet

# Set custom timeouts
aikit llm -m gpt-4o -p "Long task" --timeout 300 --connect-timeout 30
```

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--model` | `-m` | Model to use | **Required** |
| `--url` | `-u` | OpenAI-compatible API base URL | `https://api.openai.com/v1` |
| `--prompt` | `-p` | Prompt text | One of `--prompt` or `--prompt-file` required |
| `--prompt-file` | | Read prompt from file | One of `--prompt` or `--prompt-file` required |
| `--system` | | System prompt | Omitted |
| `--stream` | | Enable streaming output | `false` |
| `--out` | `-o` | Write output to file | Omitted |
| `--format` | | Output format: `text` or `json` | `text` |
| `--max-tokens` | | Maximum tokens in response | Omitted |
| `--temperature` | | Sampling temperature | Omitted |
| `--top-p` | | Top-p sampling | Omitted |
| `--api-key-env` | | Environment variable holding API key | `OPENAI_API_KEY` |
| `--timeout` | | Request timeout in seconds | `120` |
| `--connect-timeout` | | Connection timeout in seconds | `10` |
| `--usage` | | Usage output mode: `stderr`, `json`, `none` | `stderr` (text) / `json` (json) |
| `--quiet` | | Suppress non-essential output | `false` |

`--model` is required. Exactly one of `--prompt` or `--prompt-file` must be provided.

**`--format` modes:**
- `text` (default): Prints the LLM response as plain text.
- `json`: Prints a JSON envelope wrapping the full response object (including metadata).

**`--usage` modes:**
- `stderr` (default for text format): Prints token usage to stderr after the response.
- `json` (default for json format): Appends a JSON usage object to stdout after the response.
- `none`: Suppresses usage output entirely.

**`--stream` behavior:**
When `--stream` is enabled, the command connects to the SSE streaming endpoint and prints tokens incrementally. In `--format json` mode, each chunk is emitted as a JSON delta event. A final `[DONE]` event signals stream completion.

**API key resolution:**
The command reads the API key from the environment variable specified by `--api-key-env` (default: `OPENAI_API_KEY`). If the variable is not set, the command exits with an authentication error.

Run `aikit llm --help` for the authoritative option reference.

## Troubleshooting

### Common Issues

**"GitHub token required"**
```bash
# Set your GitHub token
export GITHUB_TOKEN="ghp_your_token_here"

# Or use GitHub CLI
gh auth login
```

**"AI agent CLI not found"**
```bash
# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Verify installation
aikit check
```

**"Package validation failed"**
```bash
# Check aikit.toml syntax
cat aikit.toml

# Validate package
aikit package build --debug
```

### Debug Mode

Enable debug output for detailed troubleshooting:

```bash
aikit --debug <command>
```

This provides verbose output including:
- API requests and responses
- File operations
- Validation steps
- Error stack traces

```bash
# General help
aikit --help
```
