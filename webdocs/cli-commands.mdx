# CLI Commands

AIKit provides a comprehensive command-line interface for managing AI agent packages, templates, and development workflows. This guide covers all available commands with detailed usage examples.

## Command Overview

AIKit organizes functionality into logical command groups:

### Core Commands
- **`init`** - Initialize new Spec-Driven Development projects
- **`run`** - Run a coding agent with a prompt (supports streaming events)
- **`llm`** - Invoke an LLM via OpenAI-compatible API (supports streaming and JSON output)
- **`check`** - Validate installed tools and AI agent CLIs
- **`version`** - Display version information

### Package Management
- **`install`** - Install packages from GitHub URLs
- **`update`** - Update installed packages to latest versions
- **`remove`** - Remove installed packages
- **`list`** - Display installed packages

### Package Development
- **`package init`** - Initialize new package with aikit.toml
- **`package build`** - Build package for distribution
- **`package publish`** - Publish package to registry

### Release Management
- **`release`** - Create GitHub releases with package files

## Global Options

All commands support these global options:

```bash
# Enable debug output for troubleshooting
aikit --debug <command>

# Display version information
aikit --version
aikit -V

# Show help for any command
aikit --help
aikit <command> --help
```

## Command Details

### init

Initialize a new Spec-Driven Development project with AI agent integration.

```bash
# Create new project in subdirectory
aikit init my-project

# Initialize in current directory
aikit init --here

# Initialize for specific AI agent
aikit init my-project --ai claude

# Skip Git initialization
aikit init my-project --no-git

# Force initialization in non-empty directory
aikit init existing-dir --force
```

**Options:**
- `--ai <AGENT>` - Target AI assistant (see supported agents below)
- `--script <TYPE>` - Script type (sh or ps)
- `--here` - Initialize in current directory
- `--force` - Skip confirmation for non-empty directories
- `--no-git` - Skip Git repository initialization
- `--github-token <TOKEN>` - GitHub token for API requests
- `--skip-tls` - Skip TLS verification (not recommended)
- `--ignore-agent-tools` - Skip CLI validation for selected agent

**Supported AI Agents:**
AIKit supports 18 AI assistants across different categories:

**Primary Assistants:**
- `claude` - Claude Code by Anthropic
- `cursor` - Cursor AI IDE
- `copilot` - GitHub Copilot
- `gemini` - Google Gemini

**Additional Assistants:**
- `qwen` - Qwen Code
- `opencode` - OpenCode
- `codex` - GitHub Codex
- `windsurf` - Windsurf IDE
- `kilocode` - KiloCode
- `auggie` - Auggie CLI
- `roo` - Roo Code
- `codebuddy` - CodeBuddy CLI
- `qoder` - Qoder CLI
- `amp` - Amp
- `shai` - Shai
- `continue` - Continue (Codex)
- `bob` - Bob

**What it creates:**
- Project directory with basic structure
- `aikit.toml` configuration file
- `.aikit/` directory for internal state
- Git repository (unless `--no-git` specified)
- Agent-specific templates and scripts

**Example output:**
```bash
$ aikit init my-claude-project --ai claude
✓ Created project: my-claude-project
✓ Initialized Git repository
✓ Generated aikit.toml for Claude integration
✓ Created development scripts
✓ Project ready for Spec-Driven Development
```

### run

Run a coding agent with a prompt. The `run` command executes AI coding agents directly from the CLI, supporting buffered output, agent-native streaming, and structured NDJSON event delivery.

**Runnable agents:** `codex`, `claude`, `gemini`, `opencode`, `agent`, `auto`

```bash
# Run an agent with an inline prompt
aikit run --agent claude -p "Refactor this function for clarity"

# Read prompt from stdin
echo "Add error handling to main.rs" | aikit run --agent opencode

# Enable agent-native streaming output
aikit run --agent claude --stream -p "Summarize the project"

# Emit structured NDJSON events to stdout
aikit run --agent claude --events -p "Summarize the project"

# Combine --events and --stream for streaming-aware JSON output
aikit run --agent claude --events --stream -p "Refactor this module"
```

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--agent` | `-a` | Runnable agent key | **Required** |
| `--model` | `-m` | Model passed to the agent | Omitted unless `-m` is set; agent binary defines behavior |
| `--prompt` | `-p` | Prompt to run | Reads from stdin if omitted |
| `--yolo` | | Auto-confirm, skip checks | `false` |
| `--stream` | | Enable agent-native streaming output flags | `false` |
| `--events` | | Emit NDJSON event stream to stdout | `false` |
| `--progress` | | Display live human-readable progress on stderr (conflicts with `--events`) | `false` |
| `--debug` | | Verbose diagnostic output (global `aikit` flag; shown on `run` help) | `false` |

`-a` / `--agent` is required. `aikit run` does not use `CODING_AGENT` or `CODING_AGENT_MODEL`. Without `-m`, no model is passed through to the agent.

#### `--stream` vs `--events`

These flags are independent and can be combined:

- **`--stream`**: Tunes agent-native partial output flags (e.g. `--stream-partial-output`, `stream-json`). This is passed through as `RunOptions::stream` to the argv builder. The CLI output format is unchanged.
- **`--events`**: Switches the CLI to call `run_agent_events` and emit one JSON object per line (NDJSON) to stdout. Incremental delivery; process exit code matches child exit when no SDK error occurs.
- **Combined (`--events --stream`)**: Uses events-mode JSON output AND adds stream-partial flags for supported agents.

#### NDJSON event format

When using `--events`, each line of stdout is a JSON object:

```json
{"agent_key":"claude","seq":1,"stream":"stdout","payload":{"json_line":{"type":"progress","message":"Starting..."}}}
{"agent_key":"claude","seq":2,"stream":"stdout","payload":{"raw_line":"Processing file..."}}
{"agent_key":"claude","seq":3,"stream":"stderr","payload":{"raw_line":"Warning: large context"}}
```

Fields:
- `agent_key` - The agent that produced the event
- `seq` - Monotonically increasing sequence number
- `stream` - Source stream: `"stdout"` or `"stderr"`
- `payload` - One of: `{"json_line": <object>}`, `{"raw_line": "<string>"}`, or `{"raw_bytes": "<base64>"}`

See [aikit-sdk README](../aikit-sdk/README.md) for complete streaming API documentation.

Run `aikit run --help` for the authoritative option reference.

### llm

Invoke an LLM via an OpenAI-compatible API. Supports both streaming and non-streaming requests, JSON output format, and usage tracking.

```bash
# Simple prompt (uses OPENAI_API_KEY env var by default)
aikit llm -m gpt-4o -p "Explain Rust ownership in one paragraph"

# Read prompt from a file
aikit llm -m gpt-4o --prompt-file prompt.txt

# Stream output token by token
aikit llm -m gpt-4o -p "Write a haiku about coding" --stream

# JSON output format
aikit llm -m gpt-4o -p "What is 2+2?" --format json

# Write output to a file
aikit llm -m gpt-4o -p "Summarize this" -o summary.txt

# Use a custom API endpoint
aikit llm -m my-model -u https://my-llm.example.com/v1 -p "Hello"

# Suppress non-essential output
aikit llm -m gpt-4o -p "Quick answer" --quiet

# Set custom timeouts
aikit llm -m gpt-4o -p "Long task" --timeout 300 --connect-timeout 30
```

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--model` | `-m` | Model to use | **Required** |
| `--url` | `-u` | OpenAI-compatible API base URL | `https://api.openai.com/v1` |
| `--prompt` | `-p` | Prompt text | One of `--prompt` or `--prompt-file` required |
| `--prompt-file` | | Read prompt from file | One of `--prompt` or `--prompt-file` required |
| `--system` | | System prompt | Omitted |
| `--stream` | | Enable streaming output | `false` |
| `--out` | `-o` | Write output to file | Omitted |
| `--format` | | Output format: `text` or `json` | `text` |
| `--max-tokens` | | Maximum tokens in response | Omitted |
| `--temperature` | | Sampling temperature | Omitted |
| `--top-p` | | Top-p sampling | Omitted |
| `--api-key-env` | | Environment variable holding API key | `OPENAI_API_KEY` |
| `--timeout` | | Request timeout in seconds | `120` |
| `--connect-timeout` | | Connection timeout in seconds | `10` |
| `--usage` | | Usage output mode: `stderr`, `json`, `none` | `stderr` (text) / `json` (json) |
| `--quiet` | | Suppress non-essential output | `false` |

`--model` is required. Exactly one of `--prompt` or `--prompt-file` must be provided.

**`--format` modes:**
- `text` (default): Prints the LLM response as plain text.
- `json`: Prints a JSON envelope wrapping the full response object (including metadata).

**`--usage` modes:**
- `stderr` (default for text format): Prints token usage to stderr after the response.
- `json` (default for json format): Appends a JSON usage object to stdout after the response.
- `none`: Suppresses usage output entirely.

**`--stream` behavior:**
When `--stream` is enabled, the command connects to the SSE streaming endpoint and prints tokens incrementally. In `--format json` mode, each chunk is emitted as a JSON delta event. A final `[DONE]` event signals stream completion.

**API key resolution:**
The command reads the API key from the environment variable specified by `--api-key-env` (default: `OPENAI_API_KEY`). If the variable is not set, the command exits with an authentication error.

Run `aikit llm --help` for the authoritative option reference.

## Package Management Commands

### install

Install a package from a GitHub repository URL.

```bash
# Install from GitHub URL
aikit install https://github.com/username/package-repo

# Install specific branch
aikit install https://github.com/username/package-repo --branch develop

# Install with debug output
aikit install https://github.com/username/package-repo --debug
```

**Prerequisites:**
- Valid GitHub token: `export GITHUB_TOKEN="your-token"`
- Repository must contain `aikit.toml` file

**Example output:**
```bash
$ aikit install https://github.com/example/agent-template
✓ Cloning repository...
✓ Validating aikit.toml...
✓ Installing dependencies...
✓ Package 'agent-template' installed successfully
```

```bash

# Update with debug output
aikit update --debug
```

**Example output:**
```bash
$ aikit update

✓ agent-template: 1.0.0 → 1.2.0
✓ data-processor: 0.9.0 → 1.0.0
✓ Updated 2 packages
```

## Package Development Commands

### package init

Initialize a new package with `aikit.toml` configuration.

```bash
# Initialize package with name
aikit package init my-package

# Initialize with description and version
aikit package init my-package --description "My AI agent package" --version 1.0.0

# Initialize with author information
aikit package init my-package --author "Your Name" --description "Package description"

# Skip interactive prompts
aikit package init my-package --yes

# Initialize with debug output
aikit package init my-package --debug
```

**Options:**
- `--description <TEXT>` - Package description
- `--version <VERSION>` - Package version (default: 0.1.0)
- `--author <NAME>` - Author name
- `--yes` - Skip interactive prompts

**What it creates:**
- `aikit.toml` - Package configuration file
- Basic directory structure
- Example files and documentation

**Example output:**
```bash
$ aikit package init my-package --description "AI agent utilities"
✓ Created package: my-package
✓ Generated aikit.toml configuration
✓ Package initialized successfully
```

### package build

Build package for distribution.

```bash
# Build package
aikit package build

# Build with debug output
aikit package build --debug
```

**What it does:**
- Validates `aikit.toml` configuration
- Creates distribution artifacts
- Generates package metadata
- Prepares for publishing

**Example output:**
```bash
$ aikit package build
✓ Validating package configuration...
✓ Creating distribution artifacts...
✓ Generated package metadata
✓ Build completed: dist/my-package-1.0.0.zip
```

### package publish

Publish package to registry.

```bash
# Publish to default registry
aikit package publish

# Publish with debug output
aikit package publish --debug
```

**Prerequisites:**
- Valid GitHub token with repo permissions
- Built package (run `aikit package build` first)
- GitHub repository access

**Example output:**
```bash
$ aikit package publish
✓ Authenticating with GitHub...
✓ Uploading package artifacts...
✓ Creating GitHub release...
✓ Package published: v1.0.0
✓ Available at: https://github.com/username/my-package/releases/tag/v1.0.0
```

## Troubleshooting

### Common Issues

**"GitHub token required"**
```bash
# Set your GitHub token
export GITHUB_TOKEN="ghp_your_token_here"

# Or use GitHub CLI
gh auth login
```

**"AI agent CLI not found"**
```bash
# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Verify installation
aikit check
```

**"Package validation failed"**
```bash
# Check aikit.toml syntax
cat aikit.toml

# Validate package
aikit package build --debug
```

### Debug Mode

Enable debug output for detailed troubleshooting:

```bash
aikit --debug <command>
```

This provides verbose output including:
- API requests and responses
- File operations
- Validation steps
- Error stack traces

### Getting Help

```bash
# General help
aikit --help

# Command-specific help
aikit <command> --help

# Subcommand help
aikit package --help
aikit package build --help
```
